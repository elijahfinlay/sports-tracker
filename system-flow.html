System flow

End-to-end pipeline from schedule sync to published article

1 — Schedule sync
OSAA API poll
Twice daily — pulls games, teams, coach names for all 4 schools
Cron job
Games database
Upserts games, maps OSAA statuses, stores scores when reported
SQLite
2 — Game day triggers
Game detected today
Schedule engine fires two timed actions per game
Automatic
Photo request SMS
30 min before game — asks coach for photo + invites text-in game info
Twilio
Post-game contact
After game ends — checks coach preference (call vs text-only)
Timed
3 — Coach communication
Post-game path depends on coach preference
Voice call (default)
Retell AI calls coach — 7-question interview, 3-5 min. Retry once if no answer. Rich transcript.
Retell AI
Structured text (opt-in)
SMS with 4 specific questions (score, standouts, season, upcoming). Nudge after 2hr if no reply.
Twilio
Incoming message handler
Twilio webhook — receives photos (MMS), game info text (SMS), or both. Matches sender → contact → team → today's game.
Webhook
4 — Article generation
Claude API — write article
Transcript or text message + game context + journalism prompt → headline + body. Scales length to source quality. Flags thin sources.
Anthropic API
Operator review
Dashboard notification — edit headline/body inline, attach photo, approve or reject
Human in the loop
5 — Output
REST API
Approved articles, photos, game data — all available via bearer-token-authenticated JSON endpoints
API
News site
Pulls approved articles + photos
Consumer
Other application
Pulls any data it needs
